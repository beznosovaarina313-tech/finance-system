# Система учета личных финансов 

Объектно-ориентированное Java-приложение, реализующее ключевую бизнес-логику и транзакционную безопасность распределенной системы учета доходов и расходов. 

Проект демонстрирует проектирование доменной модели данных, формализацию альтернативных сценариев бизнес-процессов через архитектуру исключений и интеграцию с реляционной СУБД.

##  Архитектура и доменная модель данных

Система четко разграничивает хранение состояния, выполнение бизнес-логики и интеграционный слой:
1. **Сущности предметной области (Пакет `src/model/`):**
   * `Transaction` (Абстрактный класс): Базовая структура финансовой транзакции, инкапсулирующая общие атрибуты (сумма, временная метка, описание, связь с категорией).
   * `Income` и `Expense` (Полиморфные наследники): Реализуют специфичное поведение для разных направлений финансовых потоков.
   * `Category`: Модель категорийного учета. Переопределенные методы `equals()` и `hashCode()` обеспечивают нормализацию и сопоставление по уникальным ID из базы данных и нечувствительным к регистру именам.
   * `BackupData`: Структурированный контейнер для выполнения моментальных снимков состояния системы.

2. **Ядро управления бизнес-логикой (`Wallet.java`):**
   * Выступает в роли главного доменного сервиса. Координирует  вычисление аналитических метрик по месяцам/годам и контролирует выполнение системных ограничений.

3. **Слой обработки бизнес-исключений (Пакет `src/exceptions/`):**
   * Проектирование исключений (`InsufficientFundsException`, `DuplicateCategoryException`) позволяет изолировать сбои логики на уровне приложения, предотвращая некорректное поведение СУБД или аварийное завершение потоков исполнения.

##  Технологический стек
* **Язык программирования:** Java 17+ (ООП: полиморфизм, наследование, инкапсуляция, механизм Checked Exceptions)
* **Технология доступа к данным:** JDBC API (через интерфейсный класс `DatabaseManager`)
* **База данных:** PostgreSQL (реляционная модель данных, транзакционный контроль)
* **Обеспечение надежности:** Спецификация Object Serialization для создания резервных копий

##  Ключевые аналитические и инженерные решения

* **Валидация платежеспособности (Alternate Flows & Constraints):** Перед персистентным сохранением объекта `Expense` (Расход) в базу данных, доменный сервис запрашивает текущий агрегированный баланс из СУБД (`db.calculate()`). Если сумма транзакции превышает доступный лимит, выполнение основного сценария блокируется и генерируется контролируемое исключение `InsufficientFundsException`.
* **Обеспечение консистентности и идемпотентности данных:** Для предотвращения замусоривания справочников реализована двухэтапная проверка уникальности категорий: проверка внутренней коллекции в оперативной памяти совмещена с параметризованным запросом существования записи в БД (`db.categoryExists()`). При попытке дублирования выбрасывается `DuplicateCategoryException`.
* **Резервное копирование и отказоустойчивость (Reliability & Recovery):** Реализована подсистема создания слепков данных в бинарном формате `.ser`. Это закрывает нефункциональные требования (NFR) к системе в части быстрого восстановления работоспособности и миграции локального состояния учетной записи.

##  Правила развертывания и запуска

### Требования к окружению
* Установленный JDK 17 или выше
* Развернутый экземпляр СУБД PostgreSQL

### Шаг 1. Подготовка базы данных
1. Создайте базу данных с именем `auth_system` (или обновите строку подключения в вашем `DatabaseManager`).
2. Выполните SQL-скрипт инициализации для создания структуры таблиц и индексов (файл `schema.sql`).

### Шаг 2. Сборка и компиляция проекта
Вы можете скомпилировать проект через вашу IDE (IntelliJ IDEA) или воспользоваться утилитой командной строки:
```bash```
# Компиляция всех пакетов из корневой директории проекта
javac -d out src/exceptions/*.java src/model/src/database/*.java src/*.java
### Шаг 3. Запуск тестов подключения
Для проверки корректности настройки прав доступа пользователя СУБД и доступности сокетов PostgreSQL запустите утилиту валидации:
java -cp out:lib/postgresql.jar CheckSQL
### Шаг 4. Запуск основного приложения
Запустите интерактивную консоль управления финансовым ядром:

```bash```
java -cp out:lib/postgresql.jar Main

### Демонстрация структуры меню
При запуске система предоставляет интерфейс со следующими Use Cases:
Добавить доход — создание транзакции типа Income с автодобавлением/выбором категории.
Добавить расход — проведение транзакции типа Expense с обязательной пре-валидацией баланса.
Баланс — вывод текущего финансового состояния на основе агрегации данных.
Статистика — группировка и фильтрация затрат за выбранный календарный период.
Создать копию / Восстановить данные — сериализация и десериализация финансовых слепков.
